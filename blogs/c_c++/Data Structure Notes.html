    # 第一章

    程序设计=数据结构+算法

    1. 数据结构基本概念： （1）数据：对客观事物的符号表示，在计算机科学中的是指所有能输入到计算机中并被计算机程序处理的符号总称 数据元素：数据的基本单位，由数据项组成 数据项：最小单位不可分割 数据对象：性质相同的数据元素的集合 抽象数据类型（ADT）:一个数学模型以及定义在该模型上的一组操作 （抽象数据组织 + 与之相关的操作） 只关注逻辑 组成：数据对象 数据关系 基本操作集 数据类型：一个值的集合+定义在这个集合的一组操作的总称 作用：用来说明变量或表达式的取值范围、存储方式 说明能进行的操作 分类：原子类型 固定聚合类型 可变聚合类型（后面统称为结构类型） 数据结构：相互之间存在一种或多种特定关系的数据元素的集合 三要素： 逻辑结构:抽象的 存储结构:具体的 数据运算 （2）数据的逻辑结构：数据对象中的数据元素之间的相互关系，描述数据之间的逻辑关系 关系（结构）： 1.集合 2.线性结构 3.树形结构 4.图状结构或网状结构 分类：线性结构（栈 队列 堆） 非线性结构 数据的存储结构：逻辑结构的关系存储到计算机中 数据域：数据元素由若干数据项构成，数据项就是数据域 表示方法：顺序映像（顺序存储结构） 非顺序映像（链式存储结构） 存储结构：顺序存储结构 链式存储结构（结构密度） 索引存储结构 散列存储结构（Hash）

       注意：存储和存取（顺序和随机）不同，不等。

    （3）数据的操作：

    2．质算法和算法的时间复杂度： （1）算法的概念和性质 指令：能被人和计算机执行 算法概念：对特定问题求解步骤的一种描述，它是指令的有限序列 算法性质：有穷性（程序不是一个算法） 确定性（指令正确） 可行性（操作可自行） 输入（0个或多个） 输出（至少一个） 算法评价：正确性 可读性 健壮性 高效率和低存储量 （2）算法的时间效率分析 方法：事后统计法（复试上机用） 事前分析估计方法（初试必考） 时间复杂度： 语句频度（一个语句出现的次数） 通过比值n->无穷大 取极限判断两个程序的时间 用增长最快o的比较 数量级：保留最复杂的一项和它的系数（最内层） f(n)为频度之和 n是算法的规模 T（n）= O(f(n)) 例子：f(n)=3*n^2+n 则 T(n)=O(n^2) T符合加法和乘法 最坏时间复杂度 最好时间复杂度 平均时间复杂度 空间复杂度：S（n） = O(g(n)) 原地工作O（1）并不是不占用内存 malloc的计算（看表达式） 原地（为1） 函数递归调用（一次调用可以看作一个单位的空间） 计算方法： 枚举法：列举归纳，根据递归的逻辑写T（n），非递归的看，控制变量，进行计数，再求计数总和（数列的求和公式） 设t值法：设t次结束，建立与n的关系，最后用n表示t(注意：log2 n 的2有时候是默认的会不写)

    # 第二章 线性表

    **线性结构（一对一） ** (非空线性表)特点：在数据元素的非空有限集合 1.存在惟一的一个被称为第一个的数据元素 ​2.存在惟一的一个被称为最后一个的数据元素 3.除了第一个之外，集合中的每个数据元素均只有一个前驱 4.除最后一个之外，集合中每一个数据元素均只有一个后继 **线性结构（**树、网、图、集合）

    # 线性表：

    # 逻辑结构：

    定义： 具有相同数据类型的n（N>=0 0为空表）个数据元素的有限序列 判定：一对一 同类型 有限 逻辑结构 ​表示： L命名 L=(a1,a2...an) ​线性表的长度：表中含有的元素个数。可以为0 ​位序：ai是第i个数据元素，称i为数据元素ai在线性表中的位序 数组小标是从0开始的，位序是从1开始的 ​顺序表：采用顺序存储对线性表的表现，是指存储结构 ​链表：采用链式存储对线性表的表现，是指存储结构 ​​存储结构​： 1.顺序存储结构： 顺序表 2.链式存储结构： 单链表 双链表 循环链表 静态链表 （前面三个是分开申请空间malloc 最后一个由数组实现）

    # 顺序表：

    定义：把线性表中所有元素按逻辑顺序，依次存储到从指定位置开始的一块连续存储空间 特点：第一个元素的存储位置就是指定的存储位置，第i+1个元素的存储位置紧接在第i个元素的存储位置后面

    相对链表： 1、可以随机访问 2、占用存储空间连续 3、顺序表的插删，需要移动多个元素

    ​实现​：（数组或者动态数组） 每+1就地址加多一个sizeof（类型） &L=a1 &L+sizeof()=a2 结构体： 静态分配特点：数空间组大小固定 动态分配（动态分配不是链式存储）特点：动态分配，空间满了可以开辟一块更大的空间 插入最后不需要移动元素（从后往前）、删除就是覆盖（从前向后） ​时间复杂度分析​： 最好的情况：在表尾插入 O(1) 平均情况：移动平均次数 n/2 o(n)

    # 链表：

    ​定义​：每个结点内容（所储存的元素和元素之间的逻辑关系信息（指针）），逻辑密度会小于1，顺序表的等于1 ​结点​：指针域和数据域 ​相对线性表​： 1、线性表只能顺序访问（存取），链表可以随机 2、占用额外的存储空间存储元素间关系，空间利用率更低 3、存储空间不一定连续 4、链表的插入不需要移动多个元素

    ​分类标准​：链表结点所包含的指针个数、指针指向和指针连接方式 ​分类​： 1、单链表 循环链表 双向链表（存储线性表）(越往后空间需求越) 2、二叉链表 十字链表 邻接表 邻接多重表（存储非线性结构 树 图） ​特点​： 1、解决了顺序表插删需要移动大量的元素的缺点 2、引入指针域，浪费空间 3、非随机存取，查找需要重头遍历。链表的最后一个结点的指针域（next）为NULL

    ​头结点​：链表第一个元素结点前附加的一个空结点 ​头结点的存在意义​：使得链表在第一个位置上的操作和其他位置上的一样，空表和非空表统一处理 空表的偷头指针为空 ​静态链表​：一个数组形式的结构体，每个结构体就是一个结点，通过下标进行操作.next是一个数组的下标，就是一个相对指针 称为游标 =-1为结尾 借用数组实现的描述线性表的链式存储结构.它不允许扩容，需要一次性申请大量内存

    ​基本操作​： 1.InintList(&L) -> 初始化列表 2.Length(L) -> 返回线性表L的长度 3.LocateElem(L,e) -> 按值查找操作，查找e值 4.GetElem(L,i) -> 按位查找操作，查找i值 5.ListInsert(&L,i,e) -> 插入操作，i位，e数值
    6.ListDelete(&L,i,&e) -> 删除操作，并用e返回删除值 7.PrintList(L) -> 输出操作（按前后顺序输出） 8.Empty(L) -> 判空操作 9.DestroyList(&L) -> 销毁操作（释放内存）

    注意：这里引用（&是在c++中才有的），如果纯c是可以直接用指正*L 1.当需要对传入本身修改时，可选择使用 2.当数据本身比较时，为避免复制开销，可以使用

    # 第三章 栈和队

    # 栈

    ​定义​：限定在表尾（或者表头 就是一端）进行插入和删除的线性表 表头：栈底（bottom） 表尾：栈顶（top）栈底是固定的，是最先入栈的 栈的插入：入栈或压栈 栈的删除: 出栈或弹栈 溢出：上下溢 栈空：栈空的约定不是定的可以是-1 也可以是0； 顺序：后进先出，最先进去的不一定最后出去，不存在出栈顺序 顺序存储方式： 1、顺序栈：数组实现 2、共享栈：就是两个栈共享一个空间，分为左右两片空间，有top1 top2 不浪费maxsiz的空间 链式存储方式： 1、链栈：单链表实现，栈空 NET->null 不存在栈满 catalan函数： 对于堆栈，入栈n次和出栈n次后栈空 总的次数是2n次 有C(2n,n)总方式，其中合法的是出栈的次数小于等于入栈的次数

    （2n,-2） 就是出n+1 ,入n-1; 不合法的次数为 c(2n,n-1) 合法的就用 c(2n,n)-c(2n,n-1)=[1/(n+1)]*c(2n,n)

    # 队列

    定义：先进先出的线性表（允许一端插入，一端删除的操作） 入队（进队）：插入 出队（离队）：删除 循环队列：就是形成一个环状，队尾和队头指针没有绝对的前后关系 双端队列：就是两端都可以插入和删除 分为前后端,受限的双端就是，某一端只能插入或者删除 顺序的实现方式： 出队指针变化：(*Q).ftont=(*Q).ftont+1)%MaxSize 入队指针变化：(*Q).rear=(*Q).rear+1)%MaxSize 队长：rear在ftont的后面，就是rear -ftont rear在ftont的前面 MaxSize-(ftont-rear) 综合 MaxSize-(ftont-rear)%MaxSize 方式一：牺牲一个元素区分队满队空（上） 方式二：添加一个size 数据成员 方式三：添加一个tag 数据成员 删除时候队伍空 0 插入导致队满 1 链式的实现方式： 链队列：带有队头和队尾指针的链表 （就是在节点上有指针） 不存在上溢出 ，适合多次插删

    # 应用

    栈： 1.括号匹配 2.表达式的求值问题 3.递归 函数栈，递归

    队列： 1.树的层次遍历 2.图的广度优先遍历 3.FCFS（先来先服务，操作系统）

    # 第六章 树（递归）

        树：根结点 分支结点 叶子结点 （每个结点有唯一前驱）  （逻辑结构）
            空树：没有结点
            子树：
            度：一个结点下的子结点个数
            树的度是结点中度最大的度
            深度是从结点一直到根结点的长度（结点个数包括自己）
            高度是从结点向下直到叶子结点的长度（结点个数包括自己）
            双亲就是前驱结点
            祖先就是从前驱开始一直往上直到根结点
            路径长度
            树的高度和深度是从根结点到叶子结点

            m叉树和度为m的树区别
            m叉树是每个结点的度为小于m 可为0 ，可以为空树
            度为m的树是这个树中最大的度 不可为空树

            性质考点：
                    n个结点的树有n-1条边 

                    结点数：
                    结点数=总度数+1
                    高度度为h的m叉树至多有m^h-1/m-1(等比数列求和)
                    高度度为h的m叉树至少有 h个 高度h度为m的数h+m-1个结点
                    度为m的树，第i层的上至多有m^h(i-1)个结点（i从1开始）
                    n个结点的m叉树的最小高度logm(n(m-1)+1) 向上取整

            顺序存储方式：
                    双亲存储结构（结构体数组 里面有数据和双亲的指针（相对指针 （数组的下标）根结点的双亲下标是-1）
                    孩子存储结构（数据域和孩子指针域（树的度为多少就设计多少个指针） 没用的为NULL  占用内存浪费）
                    孩子兄弟存储结构（数据 和 孩子 和 兄弟）跟广义表有点像
            链式存储方式：
                    邻接表的形式存储（数组+链表（链表内存对应的孩子下标））
                    数据域 + 左右指针 + （父亲指针）

    二叉树：是n个结点的集合可为空
            两个子树是分左右的不可颠倒（是有序树）（无序树是左右子树交换树不变的树）

            斜树（线性表）：左右斜树 -->线性表是一种特殊的树

            特殊二叉树：
                    1.满二叉树：高度h,结点数为2^h-1
                            每一结点的度都为二
                            只有最后一行是叶子结点
                            按层序从1开始，结点i的左树为2i，右树为2i+1 可用i/2取整得到父节点
                    
                    2.完全的二叉树：就是可以在满二叉树的基础上去掉一些编号大（要连续）的树
                            一个度为1的结点 这个结点只能在左
                            i<=[n/2]为分支结点 i>[n/2]叶子结点 --> [n/2] 是最后一个分支结点，后面都为叶子结点
                            结点的度为0或为1就是i往后的结点都是叶子结点

                    3.二叉排序树：左子树上的所有关键字都小于根结点的关键字 右子树上的所有关键字都大于根结点的关键字（左右子树也是二叉排序树）

                    4.平衡二叉树：树上任意一结点的左右子树的深度差不超过1

            性质考点：
                    1. 度为0的结点n0（叶子结点）=度为2的结点n2+1 （由结点总数公式推导）

                    2.i层结点至多2^i-1

                    3.高度度为h的二叉树至多有 2^h-1

                    4.n个结点的m叉树的最小高度log2（n+1）{同上推理}或者 log2（n） + 1 {完全二叉树的最后一行只有一个叶子结点}

                    5.有n推出度为0 1 2 的结点个数 （分奇数偶数） 

            存储结构：      
                    顺序存储方式：(数组)
                            普通的二叉树按完全二叉树的结点编号方式进行
                    链式存储方式：(二叉链表)
                            数据域 + 左右指针 + （父亲指针）

            二叉表的遍历：(递归和栈 实现) p根 l左子树 r右子树
                    先序遍历：PLR 
                    中序遍历：LPR 非递归的实现时候，结束的条件是栈空且没有右子树
                    后序遍历：LRP 非递归的实现时候，循环注意的是右子树
                    (非递归 线索二叉树)
                    层序遍历：从左到右 从上到下 按行遍历（用队列 根入队 访问根 根左右孩子入队）
                    
                    先中后是深度遍历
                    层次是广度遍历

                    手工遍历的方法很好用（截图）就是在每个结点标记123然后按树的形状描边，得到的对应标号的排序为对应的遍历方式

        树和森林的转换 
            前面的孩子（左）兄弟（兄弟）存储法  树转二叉：就是一个左指针指向自己的第一个孩子，右指针指向自己的兄弟（如果很多个兄弟就套娃在右边）
            森林转二叉树 1.先把每一棵树转换为二叉树 2.把树归为其中一颗树下的一个孩子
            反过来同理

            树转二叉树的重要规律
                    二叉树中无右孩子的结点数 = 原树中分支结点数 + 1（由转换的核心左孩子右兄弟）

    树和森林的遍历
            树的遍历
                    只有先后序没有中序（因为无法确定中）
                    手工法还是同二叉树，只是少了个2
                    树的先序遍历顺序和转换的二叉树先序遍历的序列一样的
                    树的后序遍历顺序和转换的二叉树中序遍历的序列一样的
            
            森林的遍历
                    每颗树先序遍历然后把结果按树的顺序进行拼接
                    后序同理

                     森林的中序（书上是，叫法有不同）遍历顺序和转换的二叉树中序遍历的序列一样的
    
    树的应用
            1.并查集（图也会有） 一个集合  text1
            2.二叉排序树（BST定查插找） 
                    中序遍历的时候是一个有序的序列
                    删除是比较复杂的 （考虑最近的树会不会受到影响）
                    删除叶子结点 直接操作
                    删除单分支结点   删除结点，孩子接替结点的位置
                    删除双分支结点   找左边最右下的（就是左子树最大的结点 前驱）
                                    找右边最左下的（就是右子树最小的结点 后继）
                                    和直接前驱或者直接后继交换 在删除（前面两种情况）
                    查找效率   （i层*层结点）/N
                                最复杂o(n)(就是一边斜)
                                左右子树的高度差不超过1（平衡二叉树） 平均的查找长度为O（log2 n）
            3.平衡二叉树（AVL）
                    就是在二叉排序树的基础上进行调平衡 text2
                    插入为例子 就是保持平衡的关键是将根结点变换（注意变换后结点的大小关系和位置的变化）
                    有四种插入的模型ll rr lr rl；
                    前两个只是需要换一次根结点 ，后面的两个都需要换两次根结点，需要判断最小不平衡的子树（要计算权重）
            4.赫夫曼树(wpl)
                    就是根据最短权重路径编排的一种二叉树 所以他的最小路径是固定的，但是可以有很多的组合结果
                    构建它的方法就是，在给出的结点中找出两个最先的结点最为兄弟结点，构建一个根结点为两个结点权重和的子树
                    然后在选这些结点（根结点）最小和两个为子树，继续组合。直到全部结点用完

                    赫夫曼编码就是根据给出的权重按构建赫夫曼树的方式，按照左子树为0，右子树为1，或者反过来。
                    得到叶子上的符号的编码方式

    第九章 排序

        内部和外部排序
    时空复杂度 稳定性

    插入排序
            使用折半查找优化，注意稳定性（有相同的值在其中）

            时间复杂度O（n平方）
            空间复杂度O(1)

            稳定性
            顺序表和链表都可
    
    希尔排序
            把表按d=n/某个数 划分表格，各个子表进行排序
            重复，直接到d为1（某个数是给定的，一般是/2）

            空间复杂度O(1)
            最坏时间复杂度O(n平方)(目前无法证明)
            n在某个范围内可以是n的1.3次方
            如果d=1就是插入排序

            稳定性不太行
            只能顺序表

    冒泡排序（基于交换的排序）
            就是遍历一遍只交换其中两个需要交换的关键字的位置
            重复上面的操作，指导有序
            这样每一次遍历就将其中小的往前冒
            结束没有交换了就完成

            空间复杂度O(1)
            最坏时间复杂度O(n^2) 最好是o(n)
            平均时间复杂度O(n^2)

            注意每一次移动的次数，是在swap函数里面，一般是3次

            稳定性

            顺序表和链表都可

    快速排序（基于交换的排序） 考的比较多
            最好时间复杂度为O（nlog2n） o(n*递归的次数)  //用函数栈分析递归
            最坏时间复杂度0（n平方）
            就是不断的将表分为小于和大于某个数的两边
            跟二叉树相似 递归的次数和树的高度
            平均性能最优秀的方法
            
            不稳定的
    
    简单选择排序（选择排序）

            思想：就是找到列表一个最小（最大），一次次的扫描，向前面换
            空间复杂度O（1）
            时间复杂度O（n平方）
            不稳定的
            链表和顺序表都可

    堆排序
            建堆，堆分为两类，跟完全二叉树的相关联 就是在顺序存储（层序遍历）
                    关键是堆化，其实就是找出子中最大的或者最小和目标比较，是否满足堆的条件
            排序:先中建堆，然后把堆顶的元素（可以是最大的，也可以是最小）和末端交换，并对剩余进行堆化，循环前面的；
             时间复杂度（nlog2n）

             堆的插入：先把元素放入堆底部，让他不断上升（类似建堆的操作，比较的父亲）
             堆的删除：删除那个位置，用堆底元素，然后让他下坠
             注意：对比次数，（儿子中选择最大，在和父亲比），儿子只有一个就不用选

             不稳定

    归并排序（merge）

            多个有序，合并成一个
            归并的东西越多，比较的次数也就越多

            把一个序列看成各个小的子表，利用二路归并

            实现：用一个动态的辅助数组（会占用空间）
            递归，用树理解

            空间复杂度O（n）
            时间复杂度（nlog2n）

            稳定

    基数排序：（没有关键字比较）
            就是按位，进行收集，重复，直到最后一位
            稳定



    外部排序（上面的都是内部排序）
            数据在硬盘是按块进行存储，也是按块读写，可用归并排序进行排序
            时间开销还有读写内存的时间+内部排序+内部归并的时间
            主要的影响是来自读写
